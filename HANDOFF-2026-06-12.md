## ⚠️ GOTCHAS IMPORTANTES

- Subir no GitHub **NÃO** publica a edge function. É preciso colar o código no painel do Supabase (Edge Functions → Code → Deploy). O bundler do painel **não aceita** a linha `import "../deno-shim.d.ts";` — remover ao colar.
- Ao criar função pelo editor do painel, **conferir o slug/URL** (`.../functions/v1/desafio-lembrete-vencimento`). O padrão "hyper-action" precisa ser trocado ANTES do deploy.
- No **modo teste**, a função usa o template de "hoje" mesmo que a parcela não vença hoje — então o teste pode mostrar "hoje vence" com data futura. É esperado; no envio das 8h o texto bate com a data.
- O **n8n** (Railway: `n8n-aguaspurificadoras.up.railway.app`) cuida só de aniversariantes/contas a pagar — nada a ver com este lembrete.
