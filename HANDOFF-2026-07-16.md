## ⚠️ GOTCHAS IMPORTANTES

- **Secrets vs código**: mudar secret no Supabase aplica **na próxima invocação**, sem redeploy. Só precisa `supabase functions deploy <nome>` quando o **código** da function muda.
- **CLI do Supabase não está no PATH** — usar `npx supabase <comando>` (v2.109.1 testada). Não existe subcomando `functions logs` nessa versão do CLI; logs só pelo dashboard (exige login do usuário no navegador).
- **RLS bloqueia a `anon key`** nas tabelas `pessoas`, `desafio_participantes`, `desafio_parcelas` (retorna sempre `200` com array vazio, não erro — parece "sem dado" mas é RLS). Pra consultar essas tabelas, pedir pro usuário rodar no SQL Editor do Supabase (ele está logado) e colar o resultado — não tentar contornar com service role.
- **`message_status: "accepted"` na resposta da Meta não é garantia de entrega imediata** — é só confirmação de aceite síncrono. Entrega real pode levar 1-2 min. Antes de declarar falha, esperar e reconferir.
- **Janela de 24h**: pra testar texto livre é preciso a pessoa mandar uma mensagem pro número oficial primeiro (abre a janela). Templates aprovados não têm essa exigência.
- `_shared/whatsapp-meta.ts` já expõe `enviarTextoMeta()` e `enviarTemplateMeta()` prontos — reusar sempre, não duplicar client Meta.
